Hit a message saying you can't remove a game from your group's library? This happens when the game is being auto-contributed — WR-Games is adding it automatically because you own it and have auto-contribute turned on for that folder.

The game isn't stuck. You just need to go one level up.

## Why This Happens

When a collection folder is linked to a group with **auto-contribute** on, every game in that folder appears in the group's library automatically. If you still own the game, WR-Games blocks a direct per-game removal — it would just come straight back the next time you sync anyway.

The blocked-removal dialog routes you to the controls that actually fix it.

## What To Do Instead

<Steps>
  <Step title="Turn off auto-contribute for that folder">
    Go to your **Collection**, find the folder the game is in, and turn off **Auto-contribute** for that group. Once it's off, you can remove individual games from the group normally.
  </Step>
  <Step title="Unlink the folder from the group entirely">
    If you don't want any games from that folder contributing to the group, unlink the folder. This removes all auto-contributed games from that folder at once — not just the one you were targeting.
  </Step>
</Steps>

<Tip>
  If you no longer own the game on BGG, re-syncing your collection will remove it from your Owned folder — and auto-contribute will stop adding it to the group automatically.
</Tip>

<Note>
  Turning off auto-contribute or unlinking a folder won't affect games you've added to the group manually — only the auto-contributed ones are removed.
</Note>
